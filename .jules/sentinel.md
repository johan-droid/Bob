## 2024-06-25 - DOM Element `innerHTML` XSS from `textContent` bypass
**Vulnerability:** A cross-site scripting (XSS) vulnerability was found in the `escHtml` function in `frontend/app.js`. The function attempted to sanitize HTML by setting `textContent` on a temporary `div` and then reading its `innerHTML`. This approach successfully escapes `<`, `>`, and `&`, but it fails to escape single (`'`) and double (`"`) quotes. When the sanitized string is subsequently injected into an HTML attribute without surrounding quotes (or matching quotes), it allows an attacker to break out of the attribute context and inject arbitrary malicious scripts.
**Learning:** Utilizing DOM properties like `textContent` to sanitize strings for HTML injection is insufficient because it does not cover all characters required for safely embedding variables into HTML attributes. Browsers do not escape quotes when retrieving `innerHTML`.
**Prevention:** When dynamically rendering HTML strings containing user-controlled variables using template literals (e.g., setting `.innerHTML`), always use a robust string replacement function that explicitly escapes all dangerous characters, including `&`, `<`, `>`, `"`, and `'`.
